# 세계사 마스터맵 Visual UX Philosophy 1.0

## 1. 핵심 철학

이 앱은 텍스트 암기장이 아니다.

고등 세계사는 단어와 문장을 외우는 과목처럼 보이지만, 실제로는 시간·공간·이동·충돌·교류·확산을 이해하는 과목이다. 학생이 텍스트만 읽고 외우면 사건의 이름, 왕조명, 인물명 같은 “글자껍데기”만 기억하게 된다.

이 앱의 목표는 학생이 세계사를 다음처럼 이해하도록 돕는 것이다.

> 세계사는 읽는 것이 아니라, 보이고 연결되어야 한다.

따라서 이 앱은 학습 정보를 단순 문단으로 전달하지 않는다. 세계사 지식은 지도, 연표, 면, 선, 점, 연결망, 비교 구조를 통해 시각적으로 조직되어야 한다.

---

## 2. 디자인 목표

### 2.1 텍스트보다 구조

설명문은 필요하지만, 설명문이 화면의 주인이 되어서는 안 된다.

앱의 중심은 다음 시각 구조여야 한다.

- 세계 지도
- 시간 리본
- 사건 노드
- 교역·전파·정복 경로를 나타내는 선
- 문명권·제국·종교권을 나타내는 면
- 사건 간 인과 관계를 나타내는 연결망

학생은 사건 카드를 읽기 전에 먼저 다음을 직관적으로 느껴야 한다.

- 어디서 일어난 일인가?
- 어느 시기에 속하는가?
- 어디로 확산되었는가?
- 무엇과 연결되는가?
- 어떤 흐름 속의 사건인가?

### 2.2 미니멀하지만 고급스럽게

세계사 앱은 복잡한 정보가 많이 들어간다. 그러므로 화면은 화려함보다 정돈감이 중요하다.

권장 미감:

- 고급 교육 인포그래픽
- 미니멀한 역사 지도
- 애플식 정보 정돈감
- 오래 봐도 피로하지 않은 배색
- 텍스트보다 시각적 질서가 먼저 보이는 화면

피해야 할 방향:

- 색이 너무 많은 교재식 화면
- 긴 문단이 가득한 암기장 화면
- 장난감 같은 게임 UI
- 무거운 박물관식 장식
- 불필요한 애니메이션

## 3. 세계사 시각 언어

이 앱은 세계사를 세 가지 기본 시각 단위로 표현한다.

### 3.1 점: 사건과 도시

점은 역사적 사건, 도시, 중심지를 나타낸다.

예:

- 로마
- 장안
- 바그다드
- 콘스탄티노폴리스
- 카라코룸
- 리스본
- 베르사유

점의 역할:

- 사건의 발생 위치
- 제국이나 문명권의 중심
- 교역망의 결절점
- 시험에서 자주 나오는 핵심 지명

### 3.2 선: 이동과 연결

선은 흐름을 나타낸다.

예:

- 실크로드
- 인도양 무역로
- 이슬람의 확산
- 몽골의 팽창
- 신항로 개척
- 대서양 삼각 무역
- 제국주의 침략 경로

선의 역할:

- 교역
- 전파
- 정복
- 이주
- 영향 관계
- 원인에서 결과로 이어지는 흐름

### 3.3 면: 세력과 문화권

면은 범위를 나타낸다.

예:

- 로마 제국의 영역
- 한 제국의 영향권
- 이슬람 문명권
- 몽골 제국의 최대 영역
- 오스만 제국의 세력권
- 유럽 제국주의 식민지 범위

면의 역할:

- 제국의 영역
- 종교권
- 문명권
- 문화권
- 정치적 세력권
- 시대별 영향권

정리하면:

```txt
점 = 사건과 중심지
선 = 이동과 연결
면 = 범위와 영향권
```

## 4. 핵심 화면 구조

### 4.1 메인 화면: World History Master Map

메인 화면은 카드 목록이 아니라 “세계사 지도”가 중심이어야 한다.

권장 구조:

```txt
상단: 앱 제목 / 검색 / 모드 전환
중앙: 미니멀 세계 지도
지도 위: 사건 점, 영향권 면, 이동 경로 선
하단: 시대 리본 타임라인
오른쪽 또는 하단: 선택한 사건의 핵심 카드
```

현재 카드형 프로토타입은 유지하되, 장기적으로 지도와 연표가 화면의 중심이 되도록 개선한다.

### 4.2 사건 선택 경험

학생이 사건을 클릭하면 화면은 다음 순서로 반응해야 한다.

1. 지도에서 관련 지역이 은은하게 강조된다.
2. 사건의 중심지가 점으로 표시된다.
3. 관련 이동·교역·전파 경로가 선으로 나타난다.
4. 해당 시대가 타임라인에서 강조된다.
5. 상세 카드에는 원인·결과·시험 포인트가 정리된다.

예:

`몽골 제국과 유라시아 교류` 선택 시

- 카라코룸 또는 몽골 초원 지역 점 표시
- 유라시아 전역 영향권 면 표시
- 동서 교류 경로 선 표시
- 13세기 타임라인 강조
- 연결 사건: 원, 실크로드, 마르코 폴로, 흑사병 등 표시

## 5. 학습 모드 설계

### 5.1 지도 모드

목표:

학생이 사건의 공간적 위치와 확산 방향을 이해한다.

필수 요소:

- 세계 지도
- 지역 하이라이트
- 사건 노드
- 선과 면의 기본 표현

초기 버전에서는 정교한 GIS 지도가 아니어도 된다. 단, 추후 지역 강조, 선 연결, 영향권 표현이 가능하도록 데이터 구조를 준비해야 한다.

### 5.2 연표 모드

목표:

학생이 사건을 시대 흐름 속에 배치한다.

필수 요소:

- 고대 / 중세 / 근세 / 근대 / 현대 구분
- 사건 노드
- 선택한 사건의 시기 강조
- 같은 시기 다른 지역 사건 비교 가능성

### 5.3 연결 모드

목표:

학생이 사건을 고립된 암기 대상이 아니라 인과 관계 속에서 이해한다.

필수 요소:

- 원인 노드
- 결과 노드
- 연결 사건 노드
- 비교 사건 노드

예:

```txt
대항해 시대
→ 상업 혁명
→ 가격 혁명
→ 절대 왕정
→ 식민지 확대
```

### 5.4 비교 모드

목표:

헷갈리는 개념을 나란히 비교한다.

예:

- 마우리아 왕조 vs 굽타 왕조
- 우마이야 왕조 vs 아바스 왕조
- 르네상스 vs 종교 개혁
- 영국 혁명 vs 프랑스 혁명
- 제국주의 vs 신제국주의

비교 모드는 긴 설명문보다 2열 또는 3열 카드 구조를 우선한다.

### 5.5 자료 판별 모드

목표:

수능·내신형 자료 해석 능력을 키운다.

자료 유형:

- 지도
- 사료
- 건축물
- 유물
- 초상화
- 연표
- 교역로 그림

질문 방식:

- 어느 지역인가?
- 어느 시대인가?
- 어떤 사건과 관련 있는가?
- 어떤 키워드로 판별할 수 있는가?
- 오답 선지는 왜 틀렸는가?

## 6. 데이터 구조 확장 방향

현재 사건 데이터는 텍스트 중심이다. 앞으로 지도형 학습을 위해 다음 필드를 추가할 수 있어야 한다.

```ts
mapPoints?: {
  id: string;
  label: string;
  x: number;
  y: number;
  type: "center" | "city" | "battle" | "trade" | "capital";
}[];

mapLines?: {
  id: string;
  label: string;
  points: { x: number; y: number }[];
  type: "trade" | "conquest" | "spread" | "migration" | "influence";
}[];

mapAreas?: {
  id: string;
  label: string;
  points: { x: number; y: number }[];
  type: "empire" | "civilization" | "religion" | "tradeZone" | "influence";
}[];

timeRange?: {
  start: number;
  end?: number;
  label: string;
};

connections?: {
  targetEventId: string;
  relation: "cause" | "result" | "sameEra" | "compare" | "confusion";
  label: string;
}[];
```

초기에는 x, y 좌표를 단순 비율 좌표로 사용해도 된다.

예:

```txt
x: 0 = 지도 왼쪽 끝
x: 100 = 지도 오른쪽 끝
y: 0 = 지도 위쪽 끝
y: 100 = 지도 아래쪽 끝
```

이 방식이면 정교한 지도 라이브러리를 쓰지 않아도 미니멀한 SVG 지도 위에 사건을 표시할 수 있다.

## 7. 색상 철학

색상은 정보를 구분하기 위한 최소한의 도구로 사용한다.

### 권장 팔레트

- 배경: warm ivory / soft gray / muted parchment
- 기본 텍스트: deep charcoal
- 보조 텍스트: cool gray
- 지도 선: muted slate
- 강조 색: amber, indigo, muted teal, clay red 등 절제된 색

### 시대별 색상 예시

- 고대: muted ochre
- 중세: deep indigo
- 근세: muted teal
- 근대: clay red
- 현대: graphite / cobalt

단, 한 화면에 너무 많은 색이 동시에 나오지 않게 한다.

## 8. 타이포그래피 원칙

- 긴 문단보다 짧은 레이블을 우선한다.
- 본문은 읽기 쉽게, 제목은 정보 위계를 명확히 한다.
- 한국어 가독성이 좋은 폰트 계열을 사용한다.
- 모바일에서 글자가 너무 작아지지 않게 한다.

권장 표현 방식:

```txt
긴 설명문 X
구조화된 항목 O
지도 + 짧은 라벨 O
비교표 O
원인 → 결과 흐름 O
```

---

## 9. 콘텐츠 표현 원칙

각 사건은 다음 순서로 이해되어야 한다.

1. 위치
2. 시대
3. 핵심 요약
4. 원인
5. 결과
6. 연결 사건
7. 시험 포인트
8. 헷갈리는 개념

즉, 상세 카드도 이 순서를 따르는 것이 좋다.

학생은 먼저 “어디와 언제”를 잡고, 그다음 “왜와 결과”를 이해해야 한다.

---

## 10. 코덱스 작업 원칙

Codex가 UI를 개선할 때 지켜야 할 원칙:

1. 텍스트를 늘리는 방식으로 해결하지 않는다.
2. 지도·연표·연결 구조를 우선한다.
3. 콘텐츠를 대량 생성하지 않는다.
4. 검수 전 자료는 검수 전으로 표시한다.
5. UI는 미니멀하고 고급스럽게 유지한다.
6. 지도 기능은 처음부터 완벽한 GIS가 아니라, 확장 가능한 SVG 구조로 시작해도 된다.
7. 세계사 데이터와 UI 컴포넌트를 분리한다.
8. 지도 좌표, 선, 면, 연결 정보는 데이터 필드로 관리한다.

## 11. 다음 구현 우선순위

### Phase 1: 현재 카드형 구조 안정화

- 검색
- 필터
- 상세 패널
- 검수 상태
- 데이터 분리

### Phase 2: 미니멀 세계 지도 추가

- 지도 영역 추가
- 사건 위치 점 표시
- 선택 사건 지역 강조
- 지도 데이터 필드 준비

### Phase 3: 타임라인 리본 추가

- 시대별 리본
- 사건의 시간 위치 표시
- 선택 사건 강조

### Phase 4: 선과 면 추가

- 교역로 선
- 정복 방향 선
- 문명권 면
- 제국 영향권 면

### Phase 5: 연결망과 비교 모드

- 사건 연결 그래프
- 비교 카드
- 헷갈리는 개념 묶음

## 12. 최종 지향점

최종적으로 이 앱은 다음 질문에 답해야 한다.

> 학생이 이 앱을 열었을 때, 세계사가 텍스트 묶음이 아니라 하나의 살아 있는 지도처럼 보이는가?

그렇지 않다면 아직 부족한 것이다.

이 앱의 정체성은 다음 문장으로 요약된다.

> 글자껍데기가 아니라, 역사 구조를 익히는 세계사 앱.
