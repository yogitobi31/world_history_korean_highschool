# Map Interface Spec 1.0

## 1. 핵심 결정

세계사 마스터맵의 중심 인터페이스는 반드시 실제 세계지도여야 한다.

현재처럼 아메리카, 유럽, 아시아, 아프리카 등을 사각형 카드나 추상 박스로 표현하는 방식은 허용하지 않는다. 이런 표현은 위치, 거리, 바다, 대륙의 형태, 교역로, 정복 방향, 문명권의 실제 공간감을 전달하지 못한다.

이 앱은 세계사를 텍스트가 아니라 시공간 구조로 학습하게 만드는 앱이므로, 지도는 장식이 아니라 메인 인터페이스다.

---

## 2. 금지되는 지도 표현

다음 방식은 사용하지 않는다.

- 대륙을 사각형 블록으로 표현
- 지역을 추상적인 박스 레이아웃으로 배치
- 실제 세계지도 없이 지역명만 표시
- 지도처럼 보이지 않는 다이어그램을 지도 영역에 배치
- 임시 placeholder를 완성 지도처럼 표시

현재의 사각형 기반 `역사 지리 레이어`는 장기 방향과 맞지 않으므로 실제 세계지도 기반 UI로 교체해야 한다.

---

## 3. 필수 요구사항

### 3.1 정확한 세계지도 윤곽

지도 영역에는 실제 세계 대륙과 국가/지역 윤곽을 가진 세계지도가 보여야 한다.

권장 방식:

- SVG 기반 세계지도
- TopoJSON/GeoJSON 기반 세계지도
- Natural Earth 또는 world-atlas 계열 데이터
- React에서 오버레이 가능한 지도 컴포넌트

정교한 GIS 수준은 아니어도 되지만, 사용자가 보았을 때 명확히 “세계지도”라고 인식되어야 한다.

### 3.2 지도 중심 레이아웃

세계지도는 보조 장식이 아니라 메인 인터페이스다.

권장 화면 구조:

```txt
상단: 앱 제목 / 검색 / 모드 전환
중앙 대형 영역: 정확한 세계지도
지도 위: 사건 점, 이동 선, 영향권 면
하단: 타임라인 리본
오른쪽 또는 하단: 선택한 사건의 상세 카드
```

카드 목록이 지도보다 더 강하게 보이면 안 된다. 카드와 설명은 지도에서 선택한 내용을 보조하는 역할이어야 한다.

---

## 4. 인터랙티브 요구사항

지도는 정적인 이미지가 아니라 인터랙티브 학습 도구여야 한다.

초기 버전에서 필요한 인터랙션:

1. Hover
   - 국가/지역 또는 사건 점에 마우스를 올리면 라벨 표시
   - 관련 사건 수나 대표 사건 미리보기 표시 가능

2. Click
   - 지도 위 사건 점 또는 지역을 클릭하면 사건 선택
   - 선택된 사건이 상세 패널에 표시

3. Selected Event Highlight
   - 선택한 사건의 중심지 표시
   - 관련 지역 하이라이트
   - 관련 선/면이 있다면 강조

4. Timeline Sync
   - 사건 선택 시 하단 타임라인에서 해당 시대 강조
   - 시대 필터를 바꾸면 지도 위 사건도 함께 필터링

## 5. 지도 위 시각 요소

이 앱의 지도 시각 언어는 다음 세 가지로 구성한다.

```txt
점 = 사건, 도시, 중심지
선 = 교역, 정복, 전파, 이동
면 = 제국, 문명권, 종교권, 영향권
```

### 5.1 점 Point

예:

- 로마
- 장안
- 바그다드
- 콘스탄티노폴리스
- 카라코룸
- 리스본
- 파리

필드 예시:

```ts
mapPoints?: {
  id: string;
  label: string;
  coordinates: [number, number]; // [longitude, latitude]
  type: "center" | "city" | "battle" | "trade" | "capital";
}[];
```

가능하면 x/y 비율 좌표보다 실제 위도/경도 좌표를 우선한다. 단, SVG 오버레이 구현상 임시로 x/y 좌표가 필요할 경우 변환 함수를 둔다.

### 5.2 선 Line

예:

- 실크로드
- 인도양 무역로
- 몽골의 팽창
- 이슬람의 확산
- 신항로 개척
- 대서양 삼각 무역

필드 예시:

```ts
mapLines?: {
  id: string;
  label: string;
  coordinates: [number, number][]; // longitude/latitude path
  type: "trade" | "conquest" | "spread" | "migration" | "influence";
}[];
```

### 5.3 면 Area

예:

- 로마 제국의 영역
- 한 제국의 영향권
- 이슬람 문명권
- 몽골 제국 최대 영역
- 오스만 제국 세력권

필드 예시:

```ts
mapAreas?: {
  id: string;
  label: string;
  coordinates: [number, number][][]; // polygon rings
  type: "empire" | "civilization" | "religion" | "tradeZone" | "influence";
}[];
```

## 6. 기술 권장안

### 6.1 우선 검토

- `react-simple-maps`
- `d3-geo`
- `topojson-client`
- `world-atlas` 또는 Natural Earth 기반 데이터

### 6.2 구현 방향

권장 컴포넌트:

```txt
src/components/map/WorldMapPanel.tsx
src/components/map/MapPointLayer.tsx
src/components/map/MapLineLayer.tsx
src/components/map/MapAreaLayer.tsx
src/components/timeline/TimelineRibbon.tsx
```

데이터 위치:

```txt
src/data/worldHistoryEvents.ts
src/types/worldHistory.ts
```

지도 데이터는 가능하면 다음 중 하나를 선택한다.

1. `public/maps/world-110m.json` 같은 정적 TopoJSON 파일 사용
2. `world-atlas` 패키지의 TopoJSON 사용
3. 신뢰 가능한 CDN을 임시 사용하되, 장기적으로는 로컬 파일화

## 7. 디자인 방향

세계지도는 복잡한 지리 교재 느낌이 아니라, 미니멀하고 고급스러운 인터랙티브 인포그래픽이어야 한다.

권장 스타일:

- 바다: 매우 연한 청회색 또는 웜그레이
- 육지: 연한 스톤/베이지 톤
- 국경선: 아주 얇고 낮은 대비
- 선택 지역: 절제된 포인트 컬러
- 사건 점: 작고 명확한 노드
- 이동 선: 얇고 우아한 곡선
- 영향권 면: 반투명 레이어
- 텍스트: 최소한의 라벨만

피해야 할 스타일:

- 원색 지도
- 교재 스캔 같은 지도
- 장난감 같은 아이콘 UI
- 너무 많은 선과 색상
- 대륙을 사각형으로 단순화한 가짜 지도

---

## 8. 1차 구현 성공 기준

다음 조건을 만족하면 1차 구현 성공으로 본다.

- 화면에 실제 세계지도 윤곽이 보인다.
- 기존 사각형 placeholder 지도는 제거된다.
- 지도 영역이 화면의 핵심 인터페이스처럼 보인다.
- 사건 선택 시 지도 위에 관련 점 또는 하이라이트가 나타난다.
- 타임라인과 사건 상세 패널이 지도와 함께 연결되어 보인다.
- 학습 콘텐츠를 대량 생성하지 않는다.
- 기존 검수 상태 표시 원칙을 유지한다.

---

## 9. 코덱스에게 주는 핵심 지시

현재 사각형 기반 지도 UI는 최종 방향이 아니다. 반드시 실제 세계지도 기반 인터페이스로 교체한다.

세계지도는 이 앱의 장식이 아니라 중심이다.

사용자가 처음 앱을 열었을 때 다음 느낌을 받아야 한다.

> 아, 세계사가 지도 위에서 움직이고 연결되는구나.
